# ORM의 N+1 문제와 해결법

## N+1 문제란 뭐냐

목록을 1번 조회한 뒤, 각 항목의 연관 데이터를 가져오기 위해 N번의 추가 쿼리가 발생하는 성능 문제.  
유저가 100명이면 **총 101번의 쿼리**가 나간다.

---

## 왜 발생하는가?

ORM은 기본적으로 **Lazy Loading**(지연 로딩이라 함)을 사용한다.  
연관 데이터를 선언해두더라도, 실제로 접근하는 순간에 비로소 DB에 쿼리를 보낸다.

```python
users = session.exec(select(User)).all()  # user 테이블 컬럼만 가져옴

for user in users:
    print(user.posts)  # 이 시점에 SELECT * FROM post WHERE user_id = ? 발생
```

`user.posts`는 파이썬 속성 접근처럼 보이지만, 내부적으로는 매번 SELECT 쿼리가 숨어 있다.

---

## Relationship vs Foreign Key 역할 구분

```python
class Post(SQLModel, table=True):
    user_id: int = Field(foreign_key="user.id")              # DB에 실제 FK 생성
    user: User | None = Relationship(back_populates="posts")  # 파이썬 레벨 접근 경로
```

- **`Field(foreign_key=...)`** → DB 테이블에 실제 외래키 컬럼을 만든다.
- **`Relationship()`** → `.posts`, `.user`로 접근할 수 있는 통로를 등록할 뿐, 데이터를 미리 가져오라는 뜻이 아니다.

`Relationship()`을 안 써도 DB 구조는 동일하게 동작한다.  
다만 편의 접근(`user.posts`)이 불가능해지고, 매번 직접 쿼리를 작성해야 해서 귀찮음.

---

## 해결법: 한꺼번에 가져오기

해결 간단함. **"루프 돌면서 하나씩 가져오지 말고, 미리 한꺼번에 가져오면 됨."**

### 1. selectinload (Eager Loading) — 쿼리 2번

```python
from sqlalchemy.orm import selectinload

stmt = select(User).options(selectinload(User.posts))
users = session.exec(stmt).all()
```

```sql
SELECT * FROM user;
SELECT * FROM post WHERE user_id IN (1, 2, 3, ..., 100);
```

유저를 먼저 가져온 뒤, 그 ID들을 모아서 IN 쿼리 한 방으로 posts를 가져온다.  
**1:N 관계에서 가장 범용적이고, 기본 선택으로 추천.**한다 함 (클로드 피셜)

### 2. joinedload (JOIN Eager Loading) — 쿼리 1번

```python
from sqlalchemy.orm import joinedload

stmt = select(User).options(joinedload(User.posts))
users = session.exec(stmt).unique().all()  # JOIN이라 unique() 필수
```

```sql
SELECT user.*, post.*
FROM user
LEFT OUTER JOIN post ON user.id = post.user_id;
```

처음부터 JOIN으로 합쳐서 한 번에 가져온다.  
**N:1 또는 1:1 관계에 적합.** 1:N에선 중복 행이 많아질 수 있다.

### 3. 직접 JOIN 쿼리 작성 — 쿼리 1번

```python
stmt = (
    select(User.name, Post.title)
    .join(Post, User.id == Post.user_id, isouter=True)
)
rows = session.exec(stmt).all()
```

ORM 자동 로딩에 의존하지 않고, 필요한 컬럼만 직접 뽑는다.  
**최대 성능이 필요하거나, 특정 컬럼만 조회할 때 적합.** 

---

## 정리

| 방법 | 쿼리 수 | 적합한 경우 |
|---|---|---|
| Lazy (기본) | N+1 | 연관 데이터에 접근하지 않을 때만 |
| `selectinload` | 2 | 1:N 관계, 기본 선택으로 추천 |
| `joinedload` | 1 | N:1 또는 1:1 관계 |
| 직접 JOIN | 1 | 특정 컬럼만 필요하거나 최대 성능 필요 시 |
